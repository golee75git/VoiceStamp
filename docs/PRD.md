## 1. 제품 개요

### 1.1 한 줄 정의

**VoiceStamp**는 사진 촬영 후 **음성·텍스트로 제목·메모를 남기고**, 기기에 **스탬프(사진+메타데이터)로 저장·관리·PDF·JPEG 공유**하는 모바일 앱이다.

### 1.2 문제 정의

- 현장·점검·기록 업무에서 사진과 설명을 빠르게 남겨야 한다.
- 타이핑보다 **음성 입력**이 편한 상황이 많다.
- 나중에 **목록 조회·수정·선택 PDF 전송**이 필요하다.
- 앱 재설치·데이터 손실에 대비해 **갤러리 백업**이 있으면 좋다.

### 1.3 목표

| 목표 | 설명 |
|------|------|
| 빠른 기록 | 촬영 → 제목/메모 → 저장을 최소 탭으로 |
| 음성 우선 | 제목·메모 필드에 마이크 입력 지원 |
| 오프라인 친화 | APK 단독 설치, 로컬 SQLite·파일 저장 |
| 공유 | 선택한 스탬프를 PDF·합성 JPEG로 만들어 저장/공유 |
| 자동 제목 | 날짜·시간 즉시 표시 + 위치(구·동·건물명) 비동기 추가 |
| 안전한 삭제 | 휴지통(소프트 삭제) 후 복원·영구 삭제 |
| 갤러리 백업 | 저장 시 날짜·장소명 앨범에 사진 복사, 수정 시 폴더·앨범 이동 |
| 폴더 분류 | 앱 내부 `stamps/YYYYMMDD_장소명/` + 갤러리 동일 앨범명 |

### 1.4 비목표 (Out of Scope)

- 클라우드 동기화·멀티 유저
- 카카오 로그인·소셜 공유 SDK 직접 연동
- 지도 UI 표시
- 갤러리 사진만으로 DB 메타데이터(제목·메모) 자동 복구
- iOS 스토어 배포 (구조상 가능하나 현재 작업 중심은 Android APK)

### 2.3 주요 플로우

```
[카메라] 시스템 카메라 자동 실행(줌) · 촬영 후 저장 모달
    → [저장 모달] 장소명(날짜_장소 앨범) · 제목(날짜·시간 즉시) / 위치 확인 중…
    → 제목·메모 수정·음성 입력 · 사진 탭 → 전체 보기(버리기)
    → [저장] SQLite + stamps/YYYYMMDD_장소명/ + (APK) 갤러리 — 설정에 따라 원본만 / 캡션·워터마크만 / 둘 다
[목록] ← 카메라 · 앨범 · 휴지통/설정/선택 · 카드 조회·수정 · 길게 눌러 선택
    → [수정 모달] 저장 폴더(앨범) 표시·직접 입력·[선택] 모달 · 사진 전체 보기
    → 폴더 변경 시 **해당 스탬프만** 앱 파일·갤러리 앨범 이동
    → [PDF 만들기] → 저장 / 공유 (보고서 제목·일시·정렬·워터마크 설정 반영)
    → 「PDF·이미지 파일명」 입력 (PDF·JPEG 공통, 다장 시 _1, _2 …)
    → [이미지 저장] → (APK) 갤러리 VoiceStamp / (웹) 다운로드
    → [휴지통으로 이동] (선택 삭제) — 스크롤 위치 유지
    → [수정] 전체 보기 → 휴지통 이동 — 스크롤 위치 유지
[목록 헤더 설정] 또는 [카메라 설정 ⚙] → [설정] 폴더명 · PDF·내보내기 옵션 · 손잡이 · 휴지통 비우기
[휴지통] 항목 터치 → 목록 복원
[Android 뒤로] 목록 → 카메라 / 설정 → (진입 화면) 목록 또는 카메라 / 휴지통 → 목록 / 카메라 → 종료 확인
```

## 9. 제목·위치·저장 폴더 UX (현재 동작)

### 9.1 제목

| 단계 | 동작 |
|------|------|
| 1 | 저장 모달 열림 → **즉시** `YYYYMMDD_HHmm` 제목 표시 (`capturedAt` 고정) |
| 2 | 제목 아래 **「위치 확인 중…」** 표시 |
| 3 | GPS (캐시·타임아웃) → 카카오 API → 제목에 구·동·건물명 추가 (사용자 미수정 시) |
| 4 | 실패 시 날짜·시간 제목 유지, 로딩 안내 숨김 |

**사용자 개입 시:** 제목 직접 수정 또는 제목 마이크 시작 → 자동 위치 반영 중단.

### 9.2 저장 폴더(앨범) — `4f56b07`

| 단계 | 동작 |
|------|------|
| 1 | 모달 열림 → `YYYYMMDD` 또는 `current_site_name` 기반 `refreshStampGroupDate` |
| 2 | GPS 완료 후에도 **폴더명은 변경하지 않음** (제목만 위치 반영) |
| 3 | 저장 시 `setCurrentSiteName` — 다음 촬영 기본값 갱신 |
| 4 | 날짜가 바뀌면 **날짜 부분만** 갱신, 현장명(`OO초` 등)은 유지 |
| 5 | 사용자가 폴더 입력·**[선택]** 으로 바꾸면 그 값 사용 |

**폴더 변경 시점:** 날짜 변경(자동)·사용자 직접 입력·폴더 선택·저장 시 현장명 갱신. **GPS 이동만으로는 변경 없음.**

## 10. 알려진 제한·이슈

| 항목 | 설명 |
|------|------|
| 위치 반영 지연 | 장소 문자열은 GPS·카카오 완료 후 추가 (날짜·시간은 즉시) |
| 실내 GPS | 느리거나 실패 시 날짜·시간만 표시 |
| 건물명 | 카카오 DB에 없으면 구·동만 표시 |

| 웹 PDF | OS 인쇄 품질·용량은 브라우저 의존 |
| 갤러리 백업 | 사진만 복사; 제목·메모는 SQLite에만 있음 (합성 JPEG는 제목·메모 포함) |
| 갤러리 폴더 이동 | 수정 시 해당 스탬프만 이동; 옛 앨범에 사진이 없으면 앨범 목록에서 사라질 수 있음 |

| 사진 선택 모달 | 갤러리 **읽기** 권한 시 Android 13+ 선택 UI — 앱은 **쓰기 전용** 유지 |
| PDF 원본 | UI상 「원본」이어도 HTML 임베드는 2400px 상한 적용 |
| 휴지통 DB | 코드 롤백 후에도 `deleted_at` 컬럼은 DB에 남을 수 있음 |
| Android 뒤로 | iOS에는 하드웨어 뒤로 버튼 없음. 웹 해당 없음 |
| 목록 스크롤 | 휴지통 이동 후 카메라 갔다 재진입 시 `silent` load가 `loading` 해제 필요 (수정됨 `bfb77d8`) |
| scrollToIndex | 앵커 인덱스 방식은 앱 종료 유발 → 사용 안 함 (`953c2cd` 되돌림) |
