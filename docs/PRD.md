## 1. 제품 개요

### 1.1 한 줄 정의

**VoiceStamp**는 사진 촬영 후 **음성·텍스트로 제목·메모를 남기고**, 기기에 **스탬프(사진+메타데이터)로 저장·관리·PDF·JPEG 공유**하는 모바일 앱이다.

### 1.2 문제 정의

- 현장·점검·기록 업무에서 사진과 설명을 빠르게 남겨야 한다.
- 타이핑보다 **음성 입력**이 편한 상황이 많다.
- 나중에 **목록 조회·수정·선택 PDF 전송**이 필요하다.
- 앱 재설치·데이터 손실에 대비해 **갤러리 백업**이 있으면 좋다.

### 1.3 목표

| 목표 | 설명 |
|------|------|
| 빠른 기록 | 촬영 → 제목/메모 → 저장을 최소 탭으로 |
| 음성 우선 | 제목·메모 필드에 마이크 입력 지원 |
| 오프라인 친화 | APK 단독 설치, 로컬 SQLite·파일 저장 |
| 공유 | 선택한 스탬프를 PDF·합성 JPEG로 만들어 저장/공유 |
| 자동 제목 | 날짜·시간 즉시 표시 + 위치(구·동·건물명) 비동기 추가 |
| 안전한 삭제 | 휴지통(소프트 삭제) 후 복원·영구 삭제 |
| 갤러리 백업 | 저장 시 기기 갤러리 VoiceStamp 앨범에 사진 복사 |

### 1.4 비목표 (Out of Scope)

- 클라우드 동기화·멀티 유저
- 카카오 로그인·소셜 공유 SDK 직접 연동
- 지도 UI 표시
- 갤러리 사진만으로 DB 메타데이터(제목·메모) 자동 복구
- iOS 스토어 배포 (구조상 가능하나 현재 작업 중심은 Android APK)

### 2.3 주요 플로우

```
[카메라] 촬영
    → [저장 모달] 제목(날짜·시간 즉시) / 위치 확인 중… / 장소 반영
    → 제목·메모 수정·음성 입력 (키보드 시 스크롤)
    → [저장] SQLite + 앱 내부 파일 + (APK) 갤러리 VoiceStamp 앨범
[목록] ← 카메라 · 휴지통/선택 · 카드 조회·수정 · 길게 눌러 선택
    → [PDF 만들기] → 저장 / 공유 (보고서 제목·일시·정렬·워터마크 설정 반영)
    → [이미지 저장] → (APK) 갤러리 VoiceStamp / (웹) 다운로드
    → [휴지통으로 이동] (선택 삭제)
[목록 하단 ⚙] → [설정] 폴더명 · PDF·내보내기 옵션 · 손잡이 · 휴지통 비우기
[휴지통] 항목 터치 → 목록 복원
```

## 6. 외부 연동

### 6.1 카카오 로컬 API

| API | 용도 |
|-----|------|
| `coord2regioncode` | 좌표 → 구·동 |
| `coord2address` | 좌표 → 건물명 (`building_name`) |

**사전 조건**

- [Kakao Developers](https://developers.kakao.com) 앱 생성
- **제품 설정 → 카카오맵 → 사용 설정 ON** (로컬 API 포함)
- REST API 키 → `.env`의 `EXPO_PUBLIC_KAKAO_REST_KEY`
- APK: 빌드 시 `.env` 번들 포함 → **키 변경 후 APK 재빌드 필수**
- Vercel: Environment Variables에 동일 키 설정

### 6.2 권한 (Android)

| 권한 | 용도 |
|------|------|
| CAMERA | 촬영 |
| RECORD_AUDIO | 음성 입력 |
| ACCESS_FINE/COARSE_LOCATION | 제목 위치 |
| INTERNET | 카카오 API |
| READ/WRITE_MEDIA (또는 저장소) | 갤러리 저장 (`expo-media-library`) |

패키지명: `com.voicestamp.app`

| 위치 제목 | GPS + 카카오 | 브라우저 위치 + 카카오 |
| 휴지통 | ✅ | ✅ (DB 동일) |
| 목록 UI (기어·휴지통) | ✅ | ✅ |

## 8. PDF 화질 프리셋

| 설정 | PDF 임베드 처리 | 용도 |
|------|-----------------|------|
| 원본 | 최대 2400px, JPEG 92% | 고화질 공유 (디스크 원본은 그대로) |
| 표준 | 1600px, 75% | 일반 공유 |
| 압축 | 1024px, 55% | 용량 우선 |

스탬프 원본 사진 파일(`documentDirectory`)은 PDF 설정과 무관하게 유지한다.

## 9. 제목·위치 UX (현재 동작)

| 단계 | 동작 |
|------|------|
| 1 | 저장 모달 열림 → **즉시** `YYYYMMDD_HHmm` 제목 표시 (`capturedAt` 고정) |
| 2 | 제목 아래 **「위치 확인 중…」** 표시 |
| 3 | GPS (캐시·타임아웃) → 카카오 API → 제목에 구·동·건물명 추가 (사용자 미수정 시) |
| 4 | 실패 시 날짜·시간 제목 유지, 로딩 안내 숨김 |

**사용자 개입 시:** 제목 직접 수정 또는 제목 마이크 시작 → 자동 위치 반영 중단.

## 10. 알려진 제한·이슈

| 항목 | 설명 |
|------|------|
| 위치 반영 지연 | 장소 문자열은 GPS·카카오 완료 후 추가 (날짜·시간은 즉시) |
| 실내 GPS | 느리거나 실패 시 날짜·시간만 표시 |
| 건물명 | 카카오 DB에 없으면 구·동만 표시 |
| Gradle 캐시 | `.env` 변경 후 APK는 `clean` 빌드 필요할 수 있음 |
| 웹 PDF | OS 인쇄 품질·용량은 브라우저 의존 |
| 갤러리 백업 | 사진만 복사; 제목·메모는 SQLite에만 있음 (합성 JPEG는 제목·메모 포함) |
| PDF 원본 | UI상 「원본」이어도 HTML 임베드는 2400px 상한 적용 |
| 휴지통 DB | 코드 롤백 후에도 `deleted_at` 컬럼은 DB에 남을 수 있음 |
